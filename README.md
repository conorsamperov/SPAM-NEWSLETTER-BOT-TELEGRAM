# Telegram FloodBot (readme потом перепишу так, что сильно не пинайте кто любит красивые текста)
**Author:** @conorxz
**Status:** «сделал не парясь, работает — и ладно»

Бот, который умеет всего две вещи:
• Запоминать ваши сообщения (уже отправленные, он их тупо пересылает)
• Раз в 20 секунд пересылать его в заранее прописанные чаты.
Управляется прямо из любого чата: отвечаешь на нужное сообщение .save — бот запомнил, пишешь .флуд — поехала рассылка.
Никаких кнопок, никакого веба, никакой админ-панели.
Код по-быстрому, без БД, без нормальной архитектуры, зато открытый исходник и минимум зависимостей.

**Для кого?**
Для себя и для всех, кому нужно «разослать одинаковое сообщение в N чатов» без заморочек.

**Быстрый старт:**
1. Склонировать репу
2. Зависимости (pyrogram)
3. Получаем ключи Telegram:
 •  Заходим на my.telegram.org → API Development Tools → копируем api_id и api_hash;
 •  Если хотите использовать бота (для отслеживания статуса работы), создаём его у @BotFather и берём токен.
4. Заполняем config.py
5. Указываем чаты, куда слать, в chats.py
6. Запускаем
 •  Первый запрос Telegram спросит номер телефона и код подтверждения — это сессия пользователя, она сохранится в файл session_name.session, больше не спросит.

**Как пользоваться:**
1. В любом чате (где есть ваш пользовательский акк) находите/пишете сообщение, которое нужно разослать.
2. Отвечаете на него словом .save — бот запоминает его (до первого перезапуска бота).
3. Пишете .флуд — начинается рассылка по чатам из chats.py с интервалом 20 с между чатами.
4. Если передумали — .стоп, рассылка прервётся после текущего чата.
5. Чтобы стереть старый шаблон и сохранить новый — .удалить, затем повторите пункт 1-2.

**Если бот не нужен:**
Тут в боте только статус отслеживается, поэтому можно использовать только пользовательский акк.
Вырезаем всё лишнее:
1. В config.py убираем BOT_TOKEN = ''
2. Удаляем папку function/bot
3. В clients.py убираем строку bot_client =
4. В main.py убираете всё связанное с bot_client и в запуске поменяйте bot_client на user_client
5. Проверяем, что нет ошибок импорта — python main.py должен запуститься без бота.
